# Rule Authoring

This document explains how a Knowledgenet rules application is created. Please make sure that you read the [Concepts](concepts.md) before proceeding. We have created an example project that highlights how a typical systems is designed. Please visit the [Knowledgenet Examples Repository](https://github.com/amitchatterjee.knowledgenet-examples), download/clone the repository to follow along.

**Note**: All instructions below assumes that we are using a **Linux** system. It is possible to use Windows, MacOS and other systems; you just have to make minor changes. 

## Pre-requisites
In order to write rules, you must be conversant with Python programming and best practices. 

Install:
- Python 3.14 or higher - suggest setting up a *venv* for this project. You may want to install python as well the the python-devel packages.  
- An IDE or text editor of your choice (e.g., VSCode, PyCharm)
- Access to the Knowledgenet Examples Repository.
